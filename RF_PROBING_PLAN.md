# RF/PLL probing plan

Radio marking found on the front panel:

```text
ALCATEL RADIOTELEPHONE
Made in France
ATR 420
03/94
```

Full schematic now available:

```text
schema/SchemaATR421_HD.png
```

Schematic notes and useful crops are tracked in `SCHEMA_ANALYSIS.md`.

Goal: identify three hardware behaviors in the original firmware and reproduce
them later in standalone C firmware:

1. frequency programming;
2. receive enable;
3. transmit enable.

## Known software markers

Confirmed panel LED/control commands:

| Command | Meaning | Reverse-engineering use |
| ---: | --- | --- |
| `69h` | `ANT` green on | receive/RX marker |
| `61h` / `68h` | `ANT` orange on | transmit/TX marker |
| `60h` | `ANT` off | end of RX/TX indication or state transition |

Important direction note: incoming key code `60h` is the physical `ON/OFF` key.
Outgoing command `60h` is `ANT off`.

Relevant original-code paths are documented in `RF_RX_TX_PATHS.md`.

## Scope/logic-analyzer channels

Preferred capture: use a logic analyzer if available. A 4-channel scope is
still enough if captures are repeated.

### PLL programming lines from schematic

The schematic confirms `MN01 = MC145156P`, driven through `MN03 = 54LS09`.
Continuity tracing now also maps the three programming lines to the
CPU/expander side. Best first probing points are the W1 flexible-cable / MN03
input lines, with CPU-side equivalents shown here:

| Signal | W1 pin | CPU/expander source | MN03 pins | MC145156P destination | Why |
| --- | ---: | --- | --- | --- | --- |
| `CLK` | `8` | 8031 pin `16`, `P3.6` / `/WR` | input `2`, output `3` | pin `11`, `clock` | PLL serial clock generated by `MOVX @R0,A` |
| `DS` | `2` | 8031 pin `13`, `P3.3` | inputs `4/5`, output `6` | pin `12`, `data` | PLL serial data from `SETB/CLR P3.3` |
| `E` | `7` | 8243 pin `5`, `P43` | inputs `12/13`, output `11` | pin `13`, `EN` | PLL enable/load |
| GND | board ground | - | - | reference |

Alternative points on the MC145156P/MN03 output side:

| Signal | MC145156P2 pin | Connected point | Why |
| --- | ---: | --- | --- |
| PLL CLK | `11` | `MN03`/`54LS09` pin `3` | serial clock/control |
| PLL DATA | `12` | `MN03`/`54LS09` pin `6` | serial data/control |
| PLL EN/load | `13` | `MN03`/`54LS09` pin `11` | latch/load/control |
| PLL LD/status | `9` | coupled network to pin `20` | lock/status candidate |

The `54LS09` has open-collector outputs, so compare input-side and output-side
waveforms if polarity or edge shape is unclear.

### CPU/control bus pins

Use these when probing latch writes or comparing with standalone firmware:

| Signal | CPU pin/register | Why |
| --- | --- | --- |
| `P1.0` | 8031 pin `1` | selects 8243 `MN13` directly or `MN14` through an inverter |
| `P1.4..P1.7` | 8031 pins `5..8` | 8243 command/data nibble bus |
| `P3.5` | 8031 pin `15` | 8243 strobe/PROG used by `L03CC` |
| `P3.6` / `/WR` | 8031 pin `16` | PLL clock pulse during `MOVX @R0,A`; routine `L50B3` |
| `P3.3` | 8031 pin `13` | PLL data bit; routine `L50B3` |
| `MN13` 8243 `P43` | 8243 pin `5` | PLL enable/load candidate |
| `MN13` pin `4` | likely adjacent `P42` output | microphone enable |
| W1 `CLK/DS/E` | PLL programming bus | confirmed by schematic and continuity tracing |
| `P1.1/P1.2/P1.3/P3.4` | older sync interface candidates | still possible for another peripheral, but no longer the first PLL target |

## Capture scenario A: frequency programming

Use the original firmware.

1. Power on the station and wait until the normal channel display appears.
2. Capture W1 `CLK`, `DS`, `E` during the startup sequence.
3. Enter channel `0`; save capture as `pll_ch0_start`.
4. Enter channel `1`; save capture as `pll_ch1`.
5. Enter channel `2`; save capture as `pll_ch2`.
6. Enter channel `0` again; save capture as `pll_ch0_again`.

What to look for:

- bursts on CLK/DATA/ENB when channel changes;
- whether the bit stream differs per channel;
- which edge of ENB/load frames the word;
- whether PLL LD/status changes after the burst.

Expected outcome:

- if the bit stream changes with channel, decode it as MC145156 programming
  data;
- if it does not change on W1 `CLK/DS/E`, move probes to `P1/P3.5` latch writes
  and check whether the radio uses a fixed PLL word for configured channels.

Firmware anchor: the matching original routine is documented in
`PLL_PROGRAMMING_PATH.md`. The serial shifter is `L50B3`, reached from `L4FD9`
through `L5036`, `L505C`, or `L5089`.

## Capture scenario B: receive enable

Use the original firmware.

1. Power on and wait for normal operation.
2. Capture the transition after startup when `ANT green` becomes active.
3. If TX was active, release TX/APPEL and capture return to receive.
4. Save as `rx_enable_after_start` and `rx_enable_after_tx`.

Software markers:

```asm
L4648 -> queue 16h -> L4BAB -> 60h, optionally 69h
L4600 -> queue 11h -> L4BC7 -> 6Ch, 69h
```

RF/latch routines to correlate:

```asm
L5931 / L5937 / L5948
L5977
L4600 / L4648
```

## Capture scenario C: transmit enable

Use the original firmware first.

1. Connect a dummy load or otherwise make sure transmitting is safe/legal.
2. Capture before pressing TX/APPEL.
3. Press and hold the key that causes transmission.
4. Save the first 1-2 seconds of the transition as `tx_enable_start`.
5. Include `MN13` pin `4` in at least one repeated capture; it is the confirmed
   microphone-enable line and should separate audio/mic gating from PLL
   programming.
6. Release the key and save return as `tx_release_to_rx`.

Software markers:

```asm
L468A/L468D -> queue 0Fh -> L4BA4 -> 68h
```

RF/latch routines to correlate:

```asm
L45B3:
  P1=EEh, A=7Eh
  P1=5Eh, A=FEh
  sets 25h.6 and timing values 60h/61h/4Eh

L4DE5..L4E0E:
  P1=8Eh, A=1Eh
  P1=CEh, A=EEh

L5977:
  clears 20h.0 and 25h.6
```
